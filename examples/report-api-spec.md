### 3.3 Enforced Conventions

**Authentication.** Default `BearerJWT` (RS256/ES256). `alg: none` rejected. JWT claims `iss`, `aud`, `exp`, `nbf` validated.

**Idempotency.** All `POST`/`PUT` (and any endpoint with `requires_idempotency=true`) must include `Idempotency-Key: <uuid v4>`. Server stores key→response for 24 h; reuse with a different body yields `422 IDEMPOTENCY_KEY_REUSE`.

**Errors.** RFC 7807 `application/problem+json` with mandatory `error_code` (UPPER_SNAKE_CASE), `request_id`, `timestamp`.

**Rate limits.** `standard` = 60/min (+10 burst), `elevated` = 600/min (+50 burst), `unlimited` reserved for internal traffic. `X-RateLimit-Limit`, `X-RateLimit-Remaining`, `X-RateLimit-Reset` returned on every response.

**Versioning.** URL prefix `/v2`. Additive changes do not bump major; breaking changes do. Deprecated endpoints emit `Deprecation` and `Sunset` headers (≥ 6 months notice).

**CORS.** Explicit allowlist origin (no `*` in production). `Authorization`, `Idempotency-Key`, `X-API-Key`, `X-Request-ID` whitelisted.

5. Generate typed SDKs from this spec to keep clients in lockstep with the contract.
